Advanced Search Section

The Advanced Search is accessed via the main/side menu or via the Species section. The Species section or 'Quick Search' is the best way to search for a single species if you know the scientific name or common name of the species you want to look for. Alternatively, you can click on 'Advanced Search' to do a search using different parameters (eg. if you are looking for yellow flowered plants with 5 petals, or a butterfly with blue wings, and/or all within a particular collection or collections, and more).

Note, this search does not include "observation" parameters (eg. Tags), only species-related parameters.

Once in the Advanced Search section, you can click on any combination of 'Species name', 'Common name', 'Classification', 'Category', 'Attribute' or 'Collection' depending on what you would like to search for. You can also click on any of these more than once if your search is more involved (eg. yellow flower with 5 petals is two attributes, so click 'Attribute' twice). Type at least 3 characters in any text-field to see a list of suggested search-terms, ordered by frequency of use (most used first).

The following search items can be used in an Advanced Species Search:
  • Species Name
    This is particularly useful for searching for a specific genus or within a genus (eg. Eucalyptus). For example, if you want a list of all of the Epacris species in Victoria or a list of all the Eucalypts in a particular area (eg. Eucalypts of the Victorian Volcanic Plains) - note that in many cases you will need to change the "exact" box to "contains" (see below for more on the 'exact/contains' box).
  • Common Name
    This is used a range of situations. Examples include
    • 'you know what you saw was a willow-herb on the Volcanic Plains but not sure which it was'
    • when you enter a common name in the basic species search and it fails because there are multiple species with the same common name, you can search for the common name using the advanced search - eg. if you type in the butterfly "Banks' Brown" in the basic species search it lists three species that have that common name and the search fails - to work out what they are you can do an advanced search for "Banks' Brown". From the results of this advanced search, click on the two sub-species to work out the defining attributes of the two sub-species.
  • Classification
    Classifications are scientific classifications; eg. the two main ones are arguably 'family' and 'order'. These take the form "family: orchidaceae" (orchids), "family: asteraceae" (daisies), "order: lepidoptera" (moths and butterflies), etc. More on Classifications.
  • Category
    Categories are groups of species sharing common attributes that are recognised by a commonly used name; eg. 'parrot', 'wattle', 'butterfly', 'weed', 'rare_threatened', 'fern', 'fauna_introduced', 'extinct_vic', 'EPBC-listed', etc. A full list of Categories is available on the main Species page and relevant Categories for individual species are listed on the relevant species page (eg. see the Category entries for the Amethyst Hairstreak butterfly). More on Categories.
  • Attribute
    Attributes are the basic or main observable characteristics of the species such as "flower: yellow", "petals: 5", "leaf: prickly", "wing: green", "body: yellow", "habit: groundcover" - this is the basic information that people observe when they see the species, or things that help in narrowing down a search in NatureShare. The best attributes are minimalist (eg. "flower: yellow", "wing: blue") - see Prickly Tea-tree for a good example. More on Attributes.
  • Collection
    NatureShare Collections allow people and groups to create a list of species for specific areas of interest or land. You can search within this list. For example, you can search for purple pea flowers in the Barrm Birrm plants collection. You can also search across multiple collections to find all species recorded in a particular town (NB. this is why it is always advisable, where possible, to include the name of the nearest town/area in the Collection name). More about Collections.

  • Include/Exclude
    The default setting is 'Include', which means the search results include the parameter chosen. If you choose 'Exclude' it means those species with the parameter chosen will be excluded from the search results. As an example, many plant collections contain all plants, natives and weeds (eg. 'Plants of the Castlemaine Area'). If you wanted to look at just the native plants of the Castlemaine area you can do so by 'excluding weeds', or look in the Castlemaine area for native wattles (exclude the category 'weed') or weedy wattles (include the category 'weed').
  • Exact/Contains
    The default setting is 'Exact', which means the search will look for any species/category/etc with the exact text string entered in the search box.   • AND/OR
    "AND" will list only species that match all of the search-terms, "OR" will list species that match at least one of the search-terms. In "OR" mode, the results are displayed with a bracket that signifies how many of the search terms apply to each species - eg. (1) means the species matched on only search term, (2) means the species matched on two search terms, etc. To see how it works try the same search with AND and OR used alternately. For example, if you want to know how many species occur in two specific reserves you choose 'Reserve 1 Collection' OR 'Reserve 2 Collection' (eg. plant species in Barrm Birrm OR in Conglomerate Gully). If you want to know which species occur in both reserves you choose 'Reserve 1 Collection' AND 'Reserve 2 Collection' (eg. plant species in Barrm Birrm AND in Conglomerate Gully)
  • With/Without Photos
    As you'd guess, this allows you to limit the search to:
    • 'With or without photos' - displays all species searched for (whether or not there is a photo for that species in NatureShare) - this is the default setting,
      eg. this link is for all native plants found in the Victorian Volcanic Plain.
    • 'Only species with photos' - displays the species searched for, but only those that have a photo in NatureShare
      eg. this link is for the native plants found in the Victorian Volcanic Plain that have a photo.
    • 'Only species without photos' - displays the species searched for, but only those that have no photo in NatureShare
      eg. this link is for the native plants found in the Victorian Volcanic Plain that have no photo.
      This is of course very useful for monitoring progress in Collections over time (eg. at the time of writing of this page, the number of species with no photo in this collection was about 450, and hopefully this will reduce over time ... what is it now?).
  • Order by
    Again this is pretty much as it sounds. Display results can be ordered alphabetically by
    • Scientific name
    • Common name - noting that those species with no common name are displayed first
    • Category - noting that those species with no categories are displayed first
    • Classification - noting that those species with no classification are displayed first. This is quite useful in displaying in 'order by family'.
    • Attribute - noting that those species with no attributes are displayed first
